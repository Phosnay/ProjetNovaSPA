## 4.1. Backend

Technologie choisie : Node.js

Framework possible : Express.js / NestJS

Architecture :

- Controllers
- Services (composants métier)
- Repositories + data access layer
- Séparation stricte des couches

API REST sécurisée

Validation des entrées côté serveur

Tests unitaires (Jest)

Gestion des erreurs

## 4.2. Base de données

Modélisation JMerise obligatoire

SGBD : MySQL ou PostgreSQL

Contraintes :

- intégrité référentielle
- gestion transactions
- relations MCD → MLD → MPD

Modèle minimal :

#### **ANIMAL**

- id_animal (PK)
- nom
- type (chien, chat…)
- race
- sexe
- date_naissance
- date_arrivee
- statut (disponible / réservé / adopté / en soin / en sauvetage)
- description_comportement
- photo

#### **ADOPTANT**

- id_adoptant (PK)
- nom
- prénom
- email
- téléphone
- adresse
- date_inscription

#### **DOSSIER_ADOPTION**

Lié à un animal + un adoptant + un statut.

- id_dossier (PK)
- date_ouverture
- statut (en_cours / accepté / refusé / annulé)
- commentaire

#### **PERSONNEL**

(Bénévoles + membres du refuge → regroupés en un seul acteur minimal)

- id_personnel (PK)
- nom
- prénom
- rôle (bénévole, soigneur, admin)
- téléphone
- email

#### **SOIN**

(Historique des soins / vaccins / opérations)

- id_soin (PK)
- type_soin (vaccin, vermifuge, opération…)
- date_soin
- commentaire

#### **MISSION_SAUVETAGE**

- id_mission (PK)
- titre
- description
- localisation
- date_mission
- statut (planifiée / en cours / terminée)

## 4.3. Sécurité

- Mot de passe hashé (bcrypt)
- JWT ou Sessions sécurisées
- Protection contre :
    - injection
    - XSS
    - CSRF
- Rôle utilisateur (admin / adhérent)
